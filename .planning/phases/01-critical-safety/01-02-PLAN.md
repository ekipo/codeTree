<objective>
Add path validation to all MCP tools that accept `file_path` so agents cannot read files outside the repo root via `../` traversal or absolute paths.

Purpose: Tools currently accept any string, including `"../../../etc/passwd"`, construct `root_path / file_path`, and pass it to the indexer. The indexer reads the file. This is a security vulnerability.

Output: A `_validate_path(file_path, root_path)` helper inside `create_server()` that rejects non-relative and escaping paths, applied at the top of every relevant tool function, plus regression tests that prove rejection for actual attack paths.
</objective>

Inside `create_server(root: str)`, immediately after `root_path = Path(root)`, define this helper:

```python
def _validate_path(file_path: str | None, _root: Path = root_path) -> str | None:
    """Return an error string if file_path escapes the repo root, else None."""
    if not file_path:
        return None  # Optional paths — let downstream handle None/empty
    try:
        candidate = (_root / file_path).resolve()
        candidate.relative_to(_root.resolve())
        return None  # Within bounds
    except ValueError:
        return f"Error: path '{file_path}' is outside the repo root — access denied"
```

**Why `resolve()`:** `Path("/repo") / "../etc/passwd"` evaluates to `/etc/passwd` after resolve, which `relative_to("/repo")` then raises `ValueError` for. This handles `..`, symlinks, and absolute paths uniformly.

**Why not check `Path(file_path).is_absolute()`:** resolve() already handles this — but add an explicit absolute-path guard for clarity:

```python
def _validate_path(file_path: str | None, _root: Path = root_path) -> str | None:
    if not file_path:
        return None
    # Reject absolute paths immediately (OS-specific safety before resolve)
    if Path(file_path).is_absolute():
        return f"Error: path '{file_path}' must be relative to the repo root — access denied"
    try:
        candidate = (_root / file_path).resolve()
        candidate.relative_to(_root.resolve())
        return None
    except ValueError:
        return f"Error: path '{file_path}' is outside the repo root — access denied"
```

def _is_error(result) -> bool:
    """Return True if result is a path-security error (string or dict)."""
    if isinstance(result, dict):
        return "error" in result and ("denied" in result["error"] or "outside" in result["error"] or "Error" in result["error"])
    if isinstance(result, str):
        return "denied" in result or "outside" in result or ("Error" in result and "path" in result.lower())
    return False
