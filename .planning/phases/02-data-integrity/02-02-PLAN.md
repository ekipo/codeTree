<objective>
Wrap plugin.extract_skeleton() in build() with try/except so a crashing plugin skips one file but does not crash server initialization.

Purpose: A single malformed or huge file triggering a MemoryError, AttributeError, or any other unexpected exception inside a language plugin currently propagates out of build() and crashes the server at startup. All subsequent files in the indexing loop are never processed. After this fix, the affected file gets has_errors=True with an empty skeleton, and indexing continues.

Output:
- `src/codetree/indexer.py` — try/except added around extract_skeleton() call in build()
- `tests/test_data_integrity.py` — ROBUST-01 regression tests appended
</objective>
